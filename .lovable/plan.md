

# خطة: إضافة كارت بيانات العميل (Shipment Navigator) فوق جدول المتابعة

## الفكرة

شريط تنقل + كارت تفاصيل فوق الجدول. الموظف يتنقل بين الشحنات بأزرار سابق/تالي أو بالضغط على صف. الكارت يعرض كل بيانات العميل ويسمح بتعديل الحالة والملاحظات مباشرة مع مزامنة ثنائية مع الجدول.

## التصميم

```text
┌──────────────────────────────────────────────────────────────┐
│  [فلتر: الكل | لم تُتابع]                    ◀  3/47  ▶   │
├──────────────────────────────────────────────────────────────┤
│  👤 أحمد محمد              📞 01012345678 (رابط اتصال)      │
│  📦 AWB123456   🏷️ ORD-789   💰 350 ج.م                    │
│  📍 المعادي - شارع 9 عمارة 5 (عنوان كامل + منطقة)          │
│  📋 تفاصيل الطلب: 2x تيشيرت أسود L + 1x بنطلون              │
│  🚚 حالة الشحن: مرتجع        ⏱️ منذ 3 أيام                 │
│                                                              │
│  حالة المتابعة: [dropdown ▼]   ملاحظات: [________] [حفظ]   │
└──────────────────────────────────────────────────────────────┘
┌──────────────────────────────────────────────────────────────┐
│  الجدول (الصف المحدد مظلل + auto-scroll)                    │
└──────────────────────────────────────────────────────────────┘
```

## التغييرات

### ملف واحد: `src/components/followup/FollowupShipmentsTable.tsx`

1. **State جديد**:
   - `activeIndex` — index الشحنة الحالية في القائمة المفلترة
   - `cardFilter` — فلتر الكارت: `'all'` أو `'pending'` (لم تُتابع)
   - `cardVisible` — إظهار/إخفاء الكارت

2. **مكون `ShipmentCard`** (inline component) يُعرض فوق الجدول:
   - اسم العميل + رقم الهاتف كـ `<a href="tel:...">`
   - رقم البوليصة + كود الطلب + المبلغ
   - **العنوان الكامل** (`customer_address`) + **المنطقة** (`customer_area`)
   - **تفاصيل الطلب** (`order_details`)
   - حالة الشحن (`final_status`) + عدد الأيام
   - ملاحظات الشحن (`proc_notes`)
   - dropdown لتغيير حالة المتابعة → يستدعي `updateAction` الموجود
   - input للملاحظات → يستدعي `saveNote` الموجود
   - أزرار تنقل (◀ ▶) + عداد "3 من 47"
   - فلتر سريع (الكل / لم تُتابع بعد)

3. **تفاعل الجدول**:
   - الضغط على صف → يحدّث `activeIndex`
   - الصف المطابق يأخذ `bg-primary/10` للتظليل
   - `useRef` + `scrollIntoView` عند التنقل بالأزرار

4. **المزامنة الثنائية**: الكارت والجدول يشتركوا في نفس `shipments` state — أي تعديل من أي مكان ينعكس فوراً

لا حاجة لتعديلات في قاعدة البيانات — كل الحقول المطلوبة (`customer_address`, `customer_area`, `order_details`) موجودة في جدول `shipment_tracking`.
