

## المشكلة الجذرية

حالة "مقروء/غير مقروء/لم يتم الرد" محسوبة حالياً في الفرونت اند فقط باستخدام `useRef` (ذاكرة مؤقتة تضيع عند تحديث الصفحة) + مسح آخر 1000 رسالة. هذا يسبب:
- 254 محادثة تظهر "غير مقروء" رغم أنك فتحتها وردت عليها (لأن الحالة ضاعت بعد تحديث الصفحة)
- "لم يتم الرد" ناقصة لأنها تعتمد على `lastMessageDirection` المحسوب من عينة محدودة من الرسائل

## الحل: حالة محادثة مخزنة في قاعدة البيانات

اضافة عمود `chat_status` في جدول `conversations` بثلاث حالات:

```text
┌─────────────┐    عميل بعت رسالة    ┌─────────────┐    الموظف فتح    ┌──────────────┐    الموظف رد    ┌──────────┐
│   replied    │ ──────────────────► │   unread     │ ──────────────► │  awaiting    │ ─────────────► │ replied  │
│  (عادي)      │                     │ (غير مقروء)  │                 │  (لم يتم الرد) │                │ (تم الرد) │
└─────────────┘                     └─────────────┘                 └──────────────┘                └──────────┘
```

### التغييرات المطلوبة

**1. قاعدة البيانات (Migration)**
- اضافة عمود `chat_status` في `conversations` بقيمة افتراضية `'replied'`
- تحديث كل المحادثات الحالية: اللي آخر رسالة فيها inbound ومفيش outbound بعدها → `'unread'`، اللي فيها outbound بعد inbound → `'replied'`

**2. Edge Functions (الويبهوك)**
- `whatsapp-webhook`: عند وصول رسالة من عميل → `chat_status = 'unread'`
- `send-message` و `store-webhook`: عند إرسال رد → `chat_status = 'replied'`

**3. الفرونت اند**
- `useConversations`: قراءة `chat_status` من الداتابيز بدل حسابه محلياً، وحذف كل منطق `openedInboundRef` و `readStatusRef` و مسح الرسائل لحساب unread
- `selectConversation`: عند فتح محادثة `unread` → تحديث `chat_status` إلى `'awaiting_reply'` في الداتابيز
- `ChatConversation` interface: اضافة `chatStatus` property
- `ChatList` filters: 
  - "غير مقروء" = `chatStatus === 'unread'`
  - "لم يتم الرد" = `chatStatus === 'awaiting_reply'`
- `unreadCount` يتم حسابه من `chat_status === 'unread'` (1 أو 0) بدل العد المعقد

**4. Backfill للبيانات الحالية**
- SQL migration تحدث كل المحادثات الموجودة بناءً على آخر رسالة فعلية في جدول `messages`

### النتيجة
- الحالة محفوظة في السيرفر ومش بتضيع
- الفلاتر دقيقة 100%
- أداء أفضل (مفيش مسح 1000 رسالة في كل تحديث)
